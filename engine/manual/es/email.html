Consiguiendo que su programa de correo se comunique con POPFile

POPFile funciona como un servidor proxy. Su programa de correo habla a POPFile, el cual habla con su servidor de correo. En lugar de recibir los mensajes directamente en su programa de correo, POPFile los atrapa antes, y los estudia para decidir a qué categoría deben ir destinados.

Una vez que lo ha decidido - lo cual se lleva una fracción de segundo - POPFile hace una de dos: le añade una marca como [esta] al principio del la línea de asunto; lo que significa que programas con filtrado poco elaborado como el Microsoft Outlook Express puede repartir tu correo a donde se supone que deba ir - o, para programas de correo mas avanzados como Eudora, puede añadir una nueva cabecera como esta:

X-Text-Classification: personal

- la cual puede ser utilizada por su programa de correo como un filtro.

Sin embargo, usted necesitará hacer dos cosas para conseguir que su programa de correo se comunique con POPFile:

  1. Ajustar los valores de su programa de correo para obligarle a hablar con POPFile, y
  2. preparar sus filtros de correo para sacar provecho de esas marcas.

Primero preparemos los valores de correo. Estableceremos filtros en un momento.

Mas abajo estan las instrucciones para algunos de los programas de correo mas populares. Para otros - o para usuarios avanzados - aqui tienen las instrucciones generales:

  • Cambie el nombre del servidor POP3 en su cliente de correo a 127.0.0.1, anotando la dirección original.
  • Cambie el nombre de usuario POP3 a [direccion-original-del-servidor-POP3]:[usuario]
  • Deje la contraseña tal cual

Para cuentas que necesiten Autenticación por contraseña segura, vea Usando POPFile con: Autenticación por Contraseña Segura.

Elija su programa de correo y siga las intrucciones de la columna inferior. Si bien estas instrucciones parecen complicadas, solo le tomará un par de minutos - y es dificil equivocarse. (Por si acaso, anote sus valores originales en caso de que necesite dar marcha atras.)

Outlook

  • En Outlook elija la opción de menu Herramientas->Cuentas de correo.... Verá el diálogo para cuentas de correo. [screenshot]
  • Clic en Ver o cambiar las cuentas de correo existentes y clic Siguiente. Elija la cuenta que quiera utilizar con POPFile y clic Cambiar... Anote el nombre del servidor de correo entrante (POP3) y el nombre de Usuario. [screenshot]
  • Cambie el nombre del servidor de correo entrante (POP3) a 127.0.0.1 y tambien el nombre de Usuario con una combinación del original nombre del servidor de correo entrante (POP3) y el nombre original de Usuario separados por dos puntos. [screenshot]
  • Asi, si el nombre original del servidor de correo entrante (POP3) era my.mail.com y su nombre de usuario era jdoe, su nuevo nombre de usuario será my.mail.com:jdoe
  • No tiene porqué cambiar la contraseña.
  • Pulse Siguiente y despues Terminar.
  • Asegúrese de que POPFile está funcionando y todo el correo se repartira a traves de POPFile!
  • Necesitará asegurarse que POPFile comience cuando lo hace Windows. Si utiliza la versión con instalador para Windows, puede arrastrare el acceso directo 'Start POPFile in Background' desde el grupo POPFile en el menu de Inicio hasta el grupo Inicio. Ahora, Windows iniciará POPFile siempre.
  • ¡Importante! Si no se carga POPFile, el correo no pasará a traves de el - y por lo tanto su programa de correo no conectará con el servidor!

Outlook Express

  • En Outlook Express elija el menu Herramientas->Cuentas... Aparecerá el diálogo de Cuentas de Internet. [screenshot]
  • Elija la cuenta que quiere modificar para trabajar con POPFile y clic en Propiedades. [screenshot]
  • Elija la pestaña Servidores. Anote los nombres del servidor de Correo Entrante (POP3) y de la cuenta de correo entrante. [screenshot]
  • Cambie el nombre de servidor de Correo Entrante (POP3) a 127.0.0.1 y el Nombre de Cuenta del servidor de correo a una combinación del 'nombre original del servidor para Correo Entrante' (POP3) y el 'Nombre de cuenta original del servidor de Correo Entrante', separados por dos puntos. [screenshot]
  • Asi, si el servidor original para Correo Entrante se llamaba my.mail.com y su nombre de usuario era jdoe, su nuevo nombre de usuario será my.mail.com:jdoe
  • No necesita cambiar la contraseña.
  • Pulse OK y luego Cerrar.
  • Asegúrese de que POPFile está funcionando y todo el correo será repartido mediante POPFile!
  • Necesitará asegurarse de que POPFile arranca cuando lo hace Windows. Si está usando la versión con instalador de Windows installer, puede arrastrar el acceso directo 'Start POPFile in Background' desde el grupo de POPFile en el menu Inicio al grupo Inicio. Ahora, Windows cargará POPFile cada vez que se inicie.
  • Importante! Si POPFile no se carga, el correo no podra pasar por el - y por lo tanto su programa de correo no podra conectar!

Eudora

  • En Eudora elija la opción de menu Tools->Options... . Aparecerá el diálogo Options. [screenshot]
  • Anote el nombre de servidor Mail Server (incoming) y el Login Name. [screenshot]
  • Cambie el nombre de servidor Mail Server (incoming) a 127.0.0.1 y tambien el Login Name a una combinación del nombre de servidor original del Mail Server (incoming) y el Login Name original separados por dos puntos. [screenshot]
  • Asi, si el nombre de servidor original del Mail Server (incoming) era my.mail.com y su username era jdoe, su nuevo username será my.mail.com:jdoe
  • No necesita cambiar la contraseña.
  • Pulse OK.
  • Asegúrese de que POPFile está funcionando y todo el correo será repartido mediante POPFile!
  • Necesitará asegurarse de que POPFile arranca cuando lo hace Windows. Si está usando la versión con instalador de Windows installer, puede arrastrar el acceso directo 'Start POPFile in Background' desde el grupo de POPFile en el menu Inicio al grupo Inicio. Ahora, Windows cargará POPFile cada vez que se inicie.
  • Importante! Si POPFile no se carga, el correo no podra pasar por el - y por lo tanto su programa de correo no podra conectar!

Pegasus

  • En Pegasus seleccione la pestaña Tools->Internet Options... Receiving (POP3). Aparecerá la página para definir la cuenta POP3.
  • Bajo General settings para recibir correo usando POP3, seleccione la casilla de diálogo POP3 Host. Anote el valor actual del POP3 host.
  • Cambie el POP3 Host a 127.0.0.1
  • Bajo General settings para recibir correo usando POP3, seleccione la casilla de diálogo User name. Anote el valor actual del User name.
  • Cambie el User name a una combinación del nombre original POP3 Host y el User name original separados por dos puntos.
  • Asi, si el POP3 host original era my.mail.com y su User name era jdoe, su nuevo User name será my.mail.com:jdoe
  • No necesita cambiar la contraseña.
  • Clic el botón OK para guardar los cambios.
  • Asegúrese de que POPFile está funcionando y todo el correo será repartido mediante POPFile!
  • Necesitará asegurarse de que POPFile arranca cuando lo hace Windows. Si está usando la versión con instalador de Windows installer, puede arrastrar el acceso directo 'Start POPFile in Background' desde el grupo de POPFile en el menu Inicio al grupo Inicio. Ahora, Windows cargará POPFile cada vez que se inicie.

Bien. Ya está hecho, es hora de preparar sus filtros de correo, de modo que cuando POPFile haya hecho su trabajo, su programa de correo pueda recogerlo y poner todo en su carpeta correspondiente.

Para usuarios avanzados, tienen la opción de modificar la línea de asunto para añadir una marca de [clasificacion] o no. Está al final de la pestaña de Configuración en el IU web. Sin importar qué opción elija, la cabecera X-Text-Classification se añadirá de todas formas.

Solo elija su programa de correo y siga las intrucciones de la columna inferior. Si bien estas instrucciones parecen complicadas, solo le tomará un par de minutos - y es dificil equivocarse. (Por si acaso, anote sus valores originales en caso de que necesite dar marcha atras.)

Outlook

  • En Outlook seleccione Asistente Herramientas->Reglas... [screenshot]
  • Aparecerá el asistente de Reglas. Haga clic en Nuevo... y elija Empezar desde una regla en blanco. [screenshot]
  • Outlook puede usar la cabecera X-Text-Classification, lo que permitirá que el filtrado de correo transcurra completamente en segundo plano. Haga clic en Siguiente y elija con palabras especificas en la cabecera del mensaje.
  • Ahora haga clic el enlace resaltado en gris palabras especificas y ponga "X-Text-Classification: " seguido del nombre de una de sus categorías en la casilla que aparezca y pulse Añadir, y luego OK.
  • Haga clic en Siguiente y seleccione moverlo a la carpeta especificada.
  • Haga clic en la palabra resaltada en gris especificada y aparecerá una lista de carpetas. Elija la carpeta a la que quiere mover los mensajes de esa categoría y pulse OK.
  • Pulse OK, y luego Terminar.
  • Repita el proceso para cada una de sus categorías hasta que termine y salga.
  • ¡Ya está!
  • Una nota menor: en POPFile web interface, bajo la pestaña de Configuracion, compruebe que la opción de Modificar la Linea de Asunto está en off.

Outlook Express

  • En Outlook Express seleccione Herramientas->Reglas de Mensaje->Correo... [screenshot]
  • Aparecerá la ventana de Regla para el Correo Nuevo. [screenshot]
  • Outlook Express no puede usar la cabecera X-Text-Classification , lo que permitiría filtrar en segundo plano. Así que en su lugar, haga clic Cuando la línea Asunto contiene palabras especificas y tambien clic Moverlo a la carpeta especificada [screenshot]
  • Ahora haga clic en el enlace resaltado en azul contiene palabras especificas y aparecerá el diálogo 'Escriba Palabras Especificas' [screenshot]
  • Ponga el nombre de una de sus categorías, entre [corchetes], y pulse OK. Ahora pinche la palabra resaltada en azul especificada y aparecerá el diálogo Mover. Haga clic en la carpeta a la que quiera mover los mensajes de esta categoría y pulse OK. [screenshot]
  • Pulse OK.
  • Repita el proceso para cada una de sus categorías, y salga.
  • ¡Ya está!
  • Una nota menor: en POPFile web interface, bajo la pestaña de Configuracion, compruebe que la opción de Modificar la Linea de Asunto está en on.

Eudora

Eudora permite filtrar basandose en cualquier cabecera del mensaje, lo que está muy bien - todo puede hacerse en segundo plano, incluso sin cambiar la línea de asunto!

  • Elija el menu Tools->Filters. [screenshot]
  • Aparecerá la ventana Filters. Pinche en New para crear un filtro nuevo. [screenshot]
  • En la casilla Header escriba X-Text-Classification y en la siguiente a 'contains' ponga el nombre de una de sus categorías. No añada los corchetes. [screenshot]
  • Seleccione Transfer To en la primera casilla Action y elija el buzón al que quiere destinar los mensajes de esta categoría. [screenshot]
  • Repita el proceso para cada una de sus categorías.
  • Cierre la ventana Filters y conteste Yes cuando se le pida guardar los cambios.
  • ¡Ya está!
  • Una nota menor: en POPFile web interface, bajo la pestaña de Configuracion, compruebe que la opción de Modificar la Linea de Asunto está en off.

Pegasus

  • En Pegasus seleccione Tools->Mail Filtering Rules ->Edit New Mail Filtering Rules ->Rules Applied When Folder is Opened.
  • Aparecerá la ventana New Mail Filtering Rules.
  • Pinche el botón New Rule. Aparecerá la ventana Create a New Filtering Rule.
  • Pinche en el botón Expression. Note que utilizando el filtro Expression funcionará incluso si la la opción de POPFile para modificar la línea de Asunto está desactivada. Si usted utiliza el filtro Header, la opción de POPFile para modificar la línea de Asunto deberá activarse.
  • Seleccione la casilla de diálogo 'If this regular expression'
  • Ponga el nombre de una de sus categorías, precedido de X-Text-Classification:, por ejemplo: X-Text-Classification: spam
  • Elija los radiobotónes Headers o body
  • Elija la acción deseada en la lista desplegable de Action. Por ejemplo Move
  • Si elije una acción tal como Move o Forward, pinche el botón Set.
  • Elija la carpeta de destino o ponga el 'forward to address' para la Action elejida.
  • Pinche el botón OK para añadir la regla a la lista de reglas.
  • Repita el proceso para cada regla que necesite.
  • El orden en que se aplican las reglas de filtrado puede ajustarse mediante las flechas roja / azul (up / down) de la ventana New Mail Filter Rules.
  • Seleccione el botón Save para guardar los cambios al conjunto de reglas.
  • Seleccione el botón the OK.
  • ¡Ya está!

¡Ya está todo!... pero...

A pesar de todo esto, POPFile todavia no filtrará el correo. ¿Porqué no? No sabe para que servirán cada una de las categorías que le ha impuesto. No tiene idea de lo que significa 'spam' y 'genuino', o lo que es 'trabajo' y 'personal'. Ni siquiera sabe en qué lenguaje está tu correo - y no le importa. Necesitas entrenarlo para distinguir spam del correo ordinario, o un proyecto de otro. Harás esto mediante entrenamiento - y ya nos liaremos con eso proximamente.
